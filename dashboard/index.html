How misinformation and true content spread differently
❌ False content — Wide Star (Burst)
Misinformation spreads outward immediately. One source post reaches hundreds of users at depth 1 within minutes. The tree is flat and wide — like a star. Nobody pauses to verify, so no deep chains form.

Max depth: 1–2 · Branching: high · Cross-community: high
✅ True content — Deep Chain (Deliberate)
Verified content spreads step by step through trusted groups. Each hop is a deliberate re-endorsement. The tree is tall and narrow — like a spine. Content moves through 5–7 levels before the cascade dies out.

Max depth: 5–7 · Branching: low · Stays within communities
Why this matters: InfoShield's LP algorithm exploits this asymmetry — it suppresses cross-community edges (where false content travels) while leaving within-community edges intact (where true content flows). Structural Statistics Comparison
Each metric maps directly to a variable in the paper — this is what BiGCN learns to classify.
Paper connection: The b⁻ matrix (false content SBM) has high off-diagonal entries because false content crosses polarisation classes quickly. The b⁺ matrix (true content) has stronger diagonal entries because true content propagates within communities. These two numbers are what the LP optimiser uses to compute the dropout matrix d*.
d* Dropout Matrix — What InfoShield Does
Each cell (row u → col v) shows the probability that content is allowed to pass from community u to community v. Green = 1.0 (keep all), Red = 0.0 (suppress all). The algorithm keeps diagonal cells high (within-community flow stays intact) and suppresses off-diagonal cells (cross-community false content is blocked).
